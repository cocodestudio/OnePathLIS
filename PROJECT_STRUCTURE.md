# OnePath Lab - Project Structure & Codebase Guide

This document is designed for AI assistants (like Gemini) and developers to quickly understand the purpose of every major file and directory in this Next.js application.

## Core Stack
- **Framework:** Next.js 14 (App Router)
- **Styling:** Tailwind CSS & Radix UI (shadcn/ui)
- **Database:** Prisma ORM with SQLite (can be migrated to PostgreSQL later)
- **Authentication:** NextAuth.js

---

## Directory & File Overview

### `/prisma`
- **`schema.prisma`**: The single source of truth for the database schema. Contains models for `Lab`, `User`, `Patient`, `Test` (hierarchical: Master, Category, Sub-tests), `Report`, `ReportResult`, `Bill`, etc.
- **`seed.js`**: Script used to populate the database with initial dummy data (tests, patients, etc.).

### `/src/app` (Next.js App Router)
- **`layout.tsx`**: Root layout, wraps the app in ThemeProvider, AuthProvider, and Toaster for notifications.
- **`page.tsx`**: The public-facing landing page for the application.

### `/src/app/dashboard` (Authenticated Area)
- **`layout.tsx`**: The main dashboard shell. Contains the sidebar navigation and mobile menu.
- **`page.tsx`**: Dashboard Overview. Displays high-level analytics (Total Patients, Revenue), recent reports, and quick actions.
- **`tests/page.tsx`**: **Test Master**. The complex UI for creating/editing diagnostic packages. Handles hierarchical test structures, reference ranges, and 'Custom' value types that allow hiding numeric columns on final reports.
- **`patients/page.tsx`**: Patient Directory. Lists patients with search and date filters.
- **`patients/register/page.tsx`**: Form to register a new patient.
- **`reports/page.tsx`**: Reports Directory. Lists all generated reports. Includes dynamic filters (Search, Date, Abnormal toggle).
- **`reports/[id]/edit/page.tsx`**: **Result Entry Page**. Highly complex page where lab technicians enter results. Contains logic to hide/merge columns (Unit, Reference, Flag) if the test uses a `Custom` layout.
- **`reports/[id]/page.tsx`**: The page showing the finalized report details.
- **`billing/page.tsx`**: Financial overview, lists all invoices/bills with statuses (PAID, UNPAID, PARTIAL).

### `/src/app/api` (Backend Route Handlers)
- **`send-whatsapp/route.ts`**: API endpoint triggered when sharing a report via WhatsApp. 
- **`reports/route.ts` & `reports/[id]/route.ts`**: API for fetching, creating, and updating report data and results.
- **`analytics/route.ts`**: Aggregates data for the dashboard charts.
- **`auth/[...nextauth]/route.ts`**: NextAuth configuration and credentials provider for login handling.

### `/src/components` (Reusable UI & Core Components)
- **`report-sheet.tsx`**: **Critical Component**. Contains the `ReportSheet` and `buildReportBlocks` logic. It defines the exact DOM structure used for both the on-screen Report Preview and the final printed A4 PDF. Handles the logic for hiding/merging columns for Custom test formats, and centering department headers.
- **`print-preview-dialog.tsx`**: The modal dialog that appears when clicking "Print/View" on a report. Houses the print scaling logic (`react-to-print`), appearance toggles (Letterhead), and the "Share to WhatsApp" UI logic.
- **`ui/`**: Contains all the Shadcn UI components (Buttons, Inputs, Dialogs, Selects, etc.) generated by the shadcn CLI.

### `/src/lib` (Utilities)
- **`prisma.ts`**: Initializes and exports the global Prisma client instance to prevent connection limits during Next.js hot-reloading in development.
- **`utils.ts`**: Contains the `cn()` utility used extensively across the app for merging Tailwind classes gracefully.

## Important Development Notes for Gemini
1. **Routing:** This app uses the Next.js **App Router**. All data fetching in components should ideally use Server Components where possible, and `"use client"` should only be added when hooks or interactivity (like `useState`, `onClick`) are required.
2. **Database Queries:** Always ensure queries in `src/app/api` filter by `labId` to maintain multi-tenancy security.
3. **Styling:** Follow the existing clean, professional aesthetic using Tailwind CSS and Radix UI primitives.
4. **PDF Printing:** Any styling changes to `report-sheet.tsx` must be tested carefully, as they directly impact the layout of the physical A4 printed paper via `window.print()`.
